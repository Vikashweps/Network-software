# Security Audit Report: notifications-s16

## Общая информация
- Проект: notifications-s16
- Сервис: notifications-svc-s16 (порт 8131)
- Дата: 2026-04-30

## Результаты тестов
- Статический анализ (Bandit): просканировано 180 строк кода, найдено 0 уязвимостей.
- Чек-лист безопасности: пройдено 3 из 14 проверок.
- Основные провалы: отсутствие аутентификации, шифрования трафика, лимитирования запросов и запуска от непривилегированного пользователя.

## Найденные уязвимости
1. Критическая: Нет проверки JWT-токенов. Риск: несанкционированный доступ к API. Решение: внедрить middleware авторизации.
2. Критическая: gRPC работает без TLS. Риск: перехват трафика. Решение: включить ssl_channel_credentials.
3. Средняя: Отсутствует Rate Limiting. Риск: DoS-атаки. Решение: добавить middleware ограничения частоты запросов.
4. Средняя: Контейнер запускается от root. Риск: эскалация привилегий. Решение: добавить USER appuser в Dockerfile.
5. Низкая: Чувствительные данные хранятся без шифрования. Риск: утечка при компрометации БД. Решение: шифровать поля перед записью.

## Положительные аспекты
- Валидация входных данных через Pydantic реализована корректно.
- Секреты вынесены в переменные окружения.
- Обработка ошибок не раскрывает внутреннюю структуру сервиса.
- Код соответствует стандартам безопасности (Bandit не выявил проблем).


## Вывод
Код сервиса notifications-s16 написан качественно и не содержит уязвимостей на уровне исходников. Основные риски связаны с отсутствием механизмов аутентификации и шифрования на уровне конфигурации. Требуется доработка перед деплоем.
