# Не доверяй никому

Безопасность — это не фича, которую можно "прикрутить" в конце. Это процесс.

## Основные враги (OWASP Top 10)
1. **Broken Access Control**: Пользователь A может видеть заказы Пользователя B, просто поменяв ID в URL. (Самая частая проблема!).
2. **Cryptographic Failures**: Хранение паролей в plaintext, слабые ключи, передача данных по HTTP без S.
3. **Injection**: SQL-инъекции (уже редко благодаря ORM), Command Injection.
4. **Insecure Design**: Логические ошибки. Например, шаг восстановления пароля не спрашивает секретный вопрос.
5. **Security Misconfiguration**: Дефолтные пароли админки, включенный режим Debug в продакшене, открытые порты базы данных.

## Мобильная специфика (MASVS)
- **Хранение данных**: Нельзя хранить токены и PIN-коды в `SharedPreferences` или `UserDefaults` в открытом виде. Используйте системные хранилища ключей (Keystore/Keychain).
- **Сетевое общение**: Certificate Pinning (привязка сертификата), чтобы защититься от перехвата трафика через MitM (Man-in-the-Middle).

## Главный принцип
**Zero Trust**. Сервер не должен доверять клиенту. Всё, что пришло от клиента (заголовки, куки, тело запроса), может быть подделано. Валидируйте всё.
