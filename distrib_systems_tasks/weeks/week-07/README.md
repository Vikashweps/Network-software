# Знакомство с gRPC

## Задача
REST API всем хорош, но иногда нам нужна максимальная производительность или строгая типизация контрактов. Здесь на сцену выходит **gRPC** — фреймворк от Google для удаленного вызова процедур.
На этой неделе мы опишем наш сервис не в коде, а в специальном файле `.proto`, и сгенерируем из него сервер и клиент.

## Ваш вариант
`variants/<GROUP>/<STUDENT_ID>/week-07.json`
Вам понадобится название сервиса и метода из варианта.

## Что нужно сделать
1. **Описать контракт (.proto)**:
   - В папке `proto/` создайте файл `service.proto`.
   - Опишите в нем `service`, `rpc` метод и сообщения (`message`) для запроса и ответа.
2. **Сгенерировать код**:
   - Используйте компилятор `protoc` (или инструменты Python `grpcio-tools`), чтобы превратить `.proto` файл в Python-код (`_pb2.py` и `_pb2_grpc.py`).
3. **Реализовать сервер**:
   - Напишите класс, который наследуется от сгенерированного базового класса сервиса.
   - Реализуйте метод (Unaru RPC — один запрос, один ответ), который делает то, что требуется в варианте.
4. **Запустить**:
   - Поднимите gRPC сервер на порту 50051.

## Что сдавать
1. Ваш `service.proto`.
2. Код сервера.
3. Сгенерированные файлы (обычно их не комитят, но для учебного задания можно, чтобы нам проще было проверить).
4. Ответы на вопросы.

## Как проверить
```bash
make test WEEK=07
```
Тесты сгенерируют клиент из вашего `.proto` и попытаются вызвать ваш метод.
