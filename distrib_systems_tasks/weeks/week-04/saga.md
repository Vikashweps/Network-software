# Паттерн Saga (Сага)
Вариант: `"project_code": "sessions-s16"`
## Что такое Сага?

**Сага** — это паттерн проектирования для управления распределёнными транзакциями в микросервисной архитектуре.

Паттерн Saga представляет собой методологию разбиения сложных и длинных транзакций на более мелкие, управляемые шаги. Если один шаг падает — запускаются компенсирующие транзакции для всех предыдущих успешных шагов (в обратном порядке).

## Два подхода к реализации Саги

| Подход | Описание | Когда использовать |
|--------|----------|-------------------|
| **Оркестрация** | Есть центральный сервис-координатор, который управляет всеми шагами | Когда нужна централизованная логика и контроль |
| **Хореография** | Сервисы общаются через события (каждый слушает и реагирует) | Когда сервисы должны быть максимально независимы |

> **В нашем проекте**: Используем **Оркестрацию** — `service2.py` будет координатором Саги.

# Сага для заказа пиццы

## Шаги процесса:
1. Создать заказ (статус NEW)
2. Зарезервировать пиццу на складе (уменьшить stock_quantity)
3. Списать деньги (оплата)

## Компенсации (если оплата не прошла):
1. Вернуть пиццу на склад (увеличить stock_quantity)
2. Отменить заказ (статус CANCELLED)

## Статусы заказа:
- NEW - создан
- PAID - оплачен  
- DONE - завершен
- CANCELLED - отменен
